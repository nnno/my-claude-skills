# レポートテンプレート

以下のテンプレートに従ってレポートを出力する。

```markdown
# 脆弱性リスク評価レポート

## 概要

| 項目 | 値 |
|---|---|
| CVE ID | CVE-XXXX-XXXXX |
| 対象パッケージ | {パッケージ名} {バージョン} |
| Inspector2重要度 | {CRITICAL/HIGH/MEDIUM/LOW} |
| **実質リスクレベル（総合）** | **{Critical/High/Medium/Low/Informational}** |
| 評価日 | YYYY-MM-DD |

## CVE概要

- **CVSSスコア**: X.X
- **脆弱性の種類**: {RCE/XSS/SQLi/DoS/情報漏洩/権限昇格/その他}
- **概要**: {脆弱性の技術的説明を1-2文で}
- **修正バージョン**: {判明していれば記載}

## ソースコード分析

### 検出された存在箇所

{脆弱なパッケージ/バイナリが見つかったすべての箇所を列挙する}

#### 箇所1: {箇所の説明}（例: npmパッケージ依存）

- **検出場所**: {ファイルパス}
- **使用バージョン**: {バージョン}
- **使用コンテキスト**: {ライブラリimport / バイナリ実行 / ビルド時のみ / 推移的依存}
- **実行環境**: {本番コンテナ / ビルド環境のみ / ローカル開発のみ}
- **脆弱機能の使用**: {あり/なし — 具体的な関数・コマンド名}
- **外部入力の到達**: {あり/なし — 経路の説明}
- **リスクレベル**: {Critical/High/Medium/Low/Informational}

使用箇所:
- `path/to/file.py:42` — {使用内容の説明}

#### 箇所2: {箇所の説明}（例: コンテナ内バイナリ）

- **検出場所**: {ファイルパス（Dockerfile等）}
- **使用コンテキスト**: {説明}
- **実行環境**: {説明}
- **脆弱機能の使用**: {あり/なし}
- **外部入力の到達**: {あり/なし}
- **リスクレベル**: {Critical/High/Medium/Low/Informational}

{存在箇所が1つのみの場合は箇所1のみ記載。3つ以上ある場合は同様に追加する。}

### 検出されなかった箇所
{探索したが見つからなかったレイヤーがあれば記載（調査範囲の透明性のため）}

## リスク評価

### 総合リスクレベル: {最も高いリスクレベル}

### Inspector2重要度との差分
{Inspector2の重要度と実質リスクレベルが異なる場合、その理由を説明}

### 判定根拠
{各存在箇所の評価結果をまとめ、総合判定の根拠を記載}

## 推奨対応

### 修正方法
{具体的な修正手順。存在箇所ごとに記載。}

**パッケージアップデートの場合:**
1. {パッケージ名}を{修正バージョン}以上にアップデート
2. 変更コマンド例: `{npm install / pip install / go get 等}`
3. 破壊的変更の有無: {あり/なし。ありの場合は影響箇所を記載}

**コード修正の場合:**
1. {具体的な修正内容}
2. 修正対象ファイル: {パス}

**緩和策の場合（即時修正が困難な場合）:**
1. {WAFルール追加、入力バリデーション強化等の緩和策}

### 対応優先度
- **緊急（即時対応）**: Critical — 攻撃が容易で影響が大きい
- **高（1週間以内）**: High — 攻撃可能だが条件がある
- **中（次回リリース）**: Medium — 限定的な影響
- **低（計画的対応）**: Low/Informational — 実質的影響が小さい
```
